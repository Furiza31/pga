# 📋 Functional Requirements Document – Student Association Management Platform

This document outlines the functional requirements for a web platform designed to manage the activities of a student association.

---

## ✅ Requirements Table

| Requirement ID | Description                 | User Story                                                                                                           | Expected Behavior/Outcome                                                                                                                                                                 |
| -------------- | --------------------------- | -------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **FR001**      | Member Registration         | As a new user, I want to be able to register so I can join the association.                                          | The system should provide a registration form (name, email, password, role, etc.). An email confirmation may be included.                                                                 |
| **FR002**      | Login                       | As a user, I want to be able to log in so I can access my account.                                                   | A login form should validate credentials and redirect users based on their role (admin or member).                                                                                        |
| **FR003**      | Role Management             | As an administrator, I want to assign roles to users to manage access rights.                                        | The system should allow assigning roles (admin, member) during user creation or update.                                                                                                   |
| **FR004**      | Create Events               | As an administrator, I want to create events to schedule association activities.                                     | The admin interface should allow entering a title, description, date, and location. The event should then appear in the calendar.                                                         |
| **FR005**      | View Event Calendar         | As a member, I want to view the calendar of events so I know what’s coming up.                                       | A calendar or list view should display upcoming events with their details.                                                                                                                |
| **FR006**      | Project Management          | As an administrator, I want to create collaborative projects and assign members.                                     | Each project should include a title, description, deadline, and a list of assigned members.                                                                                               |
| **FR007**      | Project Progress Tracking   | As a member, I want to track the progress of the projects I’m involved in.                                           | The project page should display tasks, deadlines, updates, and allow comments or status changes.                                                                                          |
| **FR008**      | Internal Forum              | As a member, I want to post messages to communicate with other members.                                              | Internal forum with categories, threads, and the ability to reply to messages.                                                                                                            |
| **FR009**      | Member Management           | As an administrator, I want to view, edit, or delete members of the association.                                     | A member list with options to edit or delete each entry, accessible through the admin interface.                                                                                          |
| **FR010**      | Logout                      | As a user, I want to log out to keep my account secure.                                                              | A “Logout” button should end the session and redirect to the homepage.                                                                                                                    |
| **FR011**      | Access Control              | As a developer, I want to restrict access to private pages to logged-in users only.                                  | Any private page should redirect unauthenticated users to the login page.                                                                                                                 |
| **FR012**      | Responsive Design           | As a mobile user, I want the site to adapt to my screen.                                                             | The interface should use Bootstrap to be mobile-friendly and fully responsive.                                                                                                            |
| **FR013**      | Admin Dashboard             | As an administrator, I want a clear dashboard to manage the platform’s entities.                                     | A central admin dashboard with quick access to members, projects, events, and forum sections.                                                                                             |
| **FR014**      | Action History _(optional)_ | As an admin, I want to see a history of changes to track platform activity.                                          | A log displaying the date, user, and actions taken (optional feature if time allows).                                                                                                     |
| **FR015**      | Admin Creates Users         | As an administrator, I want to manually create user accounts to add members to the association.                      | A form in the admin panel should allow entering user info (name, email, role, initial password). An email can be sent with login details or a link to set a password.                     |
| **FR016**      | Edit User Accounts          | As an administrator, I want to be able to edit user accounts so I can update their information or change their role. | From the admin panel, the administrator should be able to edit a user’s profile: name, email, role, and optionally reset the password. Changes should be saved and reflected immediately. |
